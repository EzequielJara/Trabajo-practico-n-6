  3. ¿Como es el manejo de la variables, tipo de datos y expresiones regulares en JS?

    Respuesta 3:

    Manejo de variables:

    • Hay tres tipos de variables estas son var, let, const.
    • var: Nos permite crear variables que pueden ser modificadas.
    • let: Esta variables nos permiten modificarlas, pero no se pueden re-declarar. Tambien puede servirnos para escribir un codigo mas limpio al usar funcines internas.
    • const: Se utiliza para crear variables de solo lectura, quiere decir que una vez que se crea esta no puede ser modificada.

      Tipos de datos:

      • Undefined:Es cuando a una variable no se le asigna un valor como tal, estas son de tipo Undefined.
      • Boolean: Este puede presentar dos valores como tal. Es decir, comprueban que una condicion sea "esto o esto". Por ejemplo: verdaro o falso
      • Number: Este tipoo de dato es numerico y consta de 64 bits. Tiene tres valores: +Infinity, -Infinity y NaN ("Not a Number" o No es un número).
      • String: Se utiliza para crear cadenas, el primer elemento tiene la posición 0 y el segundo la posición 1.
      • Bigint: Se utiliza para almacenar y operar grandes numeros de una forma segura.
      • Symbol: Este es un valor unico y no se puede cambiar o sobreescribir y se puede utilizar como la clave una propiedad de objeto.
      • Object:Se utilizan para almacenar varios datos y colecciones de entidades más complejas asociadas con nombres clave. En JavaScript, los objetos impregnan casi todos los aspectos del lenguaje.

  4. ¿Como son todas las formas de declaracion de funcion de JS?

    Respuesta 4:

      Function declaration

      Este tipo de funcion es la mas sencilla
      function hola(nombre){
      console.log(`Hola ${nombre}.`)
      }
      hola('Exequiel');

      Function expression

      La única diferencia con la anterior es que la definición de nuestra nueva función
      no comienza por instrucción function y el nombre de la función es opcional. Asignarla a una variable como un objeto
      const ejemplo = function(...) { ... }
      Crear un método en un objecto
      {
      ...
      'suma': function(...) { ... },
      ...
      } Utilizar la función como un callback
      .cb(function(...) { ... })

      IIFE (Immediately Invoked Function Expression)

      Para crear una función de este tipo deberemos crearla en un operador de agrupación ().
      Seguido de (), lo que posibilitará el interpretado directamente en el motor de JavaScript.
      ( function () {
      let nombre = 'Exequiel'
      return `Hola ${nombre}`
      })()

      Shorthand method definition

      Para crear este tipo de función debemos asignar un nombre de función seguido de una lista de parámetros entre paréntesis
      y los símbolos de llaves para delimitar el cuerpo de las instrucciones.
      const poblacion = {
      personas: [],
      add(...personas) {
      this.personas.push(...personas);
      }
      saluda(index) {
      return `Hola soy ${this.personas[index]}`;
      }
      }
      poblacion.add('Exequiel', 'Ezequiel', 'Guada');
      poblacion.saluda(1);

      Arrow function

      Además de la sintaxis que es diferente a las anteriores este tipo de funciones tienen las siguientes características.
      Las "arrow function" no crean su propio contexto al ejecutarse.
      Al contrario que las "function expression" o las "function declaration" que crea su propio contexto.
      Las "arrow function" son anónimas.
      El objeto arguments no se encuentra en el contexto de la función.
      Si al definir la función no utilizamos el símbolo de las llaves.
      La función devolverá como resultado de la función el resultado de la ejecución de la instrucción que hayamos indicado.
      const saluda = (nombre) => {
      return `Hola ${nombre}`;
      }
      console.log(saluda('Exequiel'))
      // Vamos a compactar un poco más el código.
      const saluda2 = (nombre) => `Hola ${nombre}`;
      console.log(saluda2('Exequiel));

      Generator function

      La sintaxis de estas funciones es igual a las "function declaration" o "function expression".
      Sólo debemos utilizar el símbolo * al comienzo de la definición de nuestra función.
      Otra diferencia de estas funciones es que siempre retorna un objeto con la siguiente estructura cada vez que invoquemos a la función next().
      function *generatorFunction()
      {
      console.log('Ejemplo generator Function.');
      // yield. Salimos de la función.
      // La propiedad value es igual a 'Un kit kat'
      yield 'Un kit kat';
      console.log(' ¿ Continuamos ?');
      // yield. Volvemos a salir de la función.
      // La propiedad value es igual a 'Fin del ejemplo'
      yield 'Fin del ejemplo';
      }
      const gen = generatorFunction();
      console.log(gen.next().value);
      console.log('La función se encuentra pausada');
      console.log(gen.next().value);
      console.log(gen.next().value);
      // El resultado de la ejecución sería el siguiente:
      // => Ejemplo generator Function.
      // => Un kit kat
      // => La función se encuentra pausada
      // => ¿ Continuamos ?
      // => Fin del ejemplo
      // => undefined

      Function constructor

      Las funciones en Javascript, son funciones, y poseen el constructor Function.
      Por ello podremos crear una variable que invoque a un objeto Function.
      Cuando invoquemos a este objeto podremos enviar todos los argumentos que deseemos.
      Los primeros n argumentos serán los parámetros de nuestra función y el último argumento será el código de nuestra función.
      const sumaFunction = new Function('numero_1', 'numero_2',
      'return numero_1 + numero_2'
      );
      // Ejecutamos nuestra función.
      console.log(sumaFunction(10, 15))

  5. ¿Cuales son los operadores y conversion entre tipos?

    Respuesta 5:

    Nombre Operador Significado
    Asignación de adición x += y x = x + y
    Asignación de resta x -= y x = x - y
    Asignación de multiplicación x *= y x = x * y
    Asignación de división x /= y x = x / y

    ToString La conversión a string ocurre cuando necesitamos la representación en forma de texto de un valor.
    Por ejemplo, alert(value) lo hace para mostrar el valor como texto.
    También podemos llamar a la función String(value) para convertir un valor a string:
    let value = true;
    alert(typeof value); // boolean
    value = String(value); // ahora value es el string "true"
    alert(typeof value); // string
    La conversión a string es bastante obvia. El boolean false se convierte en "false", null en "null", etc.
    ToNumber
    La conversión numérica ocurre automáticamente en funciones matemáticas y expresiones.
    Por ejemplo, cuando se dividen valores no numéricos usando /:
    alert( "6" / "2" ); // 3, los strings son convertidos a números
    Podemos usar la función Number(value) para convertir de forma explícita un valor a un número:
    let str = "123";
    alert(typeof str); // string
    let num = Number(str); // se convierte en 123
    alert(typeof num); // number
    La conversión explícita es requerida usualmente cuando leemos un valor desde una fuente basada en texto,
    como lo son los campos de texto en los formularios, pero que esperamos que contengan un valor numérico.
    Si el string no es un número válido, el resultado de la conversión será NaN. Por ejemplo:
    let age = Number("un texto arbitrario en vez de un número");
    alert(age); // NaN, conversión fallida

  9. ¿Cuales son todos los eventos que trabaja JS?

    Respuesta 9:

    Tipo de evento

    Nombre con prefijo on (eliminar cuando proceda)

    Descripción

    Relacionados con el ratón

    onclick

    Click sobre un elemento

    ondblclick

    Doble click sobre un elemento

    onmousedown

    Se pulsa un botón del ratón sobre un elemento

    onmouseenter

    El puntero del ratón entra en el área de un elemento

    onmouseleave

    El puntero del ratón sale del área de un elemento

    onmousemove

    El puntero del ratón se está moviendo sobre el área de un elemento

    onmouseover

    El puntero del ratón se sitúa encima del área de un elemento

    onmouseout

    El puntero del ratón sale fuera del área del elemento o fuera de uno de sus hijos

    onmouseup

    Un botón del ratón se libera estando sobre un elemento

    contextmenu

    Se pulsa el botón derecho del ratón (antes de que aparezca el menú contextual)

    Relacionados con el teclado

    onkeydown

    El usuario tiene pulsada una tecla (para elementos de formulario y body)

    onkeypress

    El usuario pulsa una tecla (momento justo en que la pulsa) (para elementos de formulario y body)

    onkeyup

    El usuario libera una tecla que tenía pulsada (para elementos de formulario y body)

    Relacionados con formularios

    onfocus

    Un elemento del formulario toma el foco

    onblur

    Un elemento del formulario pierde el foco

    onchange

    Un elemento del formulario cambia

    onselect

    El usuario selecciona el texto de un elemento input o textarea

    onsubmit

    Se pulsa el botón de envío del formulario (antes del envío)

    onreset

    Se pulsa el botón reset del formulario

    Relacionados con ventanas o frames

    onload

    Se ha completado la carga de la ventana

    onunload

    El usuario ha cerrado la ventana

    onresize

    El usuario ha cambiado el tamaño de la ventana

    Relacionados con animaciones y transiciones

    animationend, animationiteration, animationstart, beginEvent, endEvent, repeatEvent, transitionend

    Relacionados con la batería y carga de la batería

    chargingchange, chargingtimechange, dischargingtimechange, levelchange

    Relacionados con llamadas tipo telefonía

    alerting, busy, callschanged, connected, connecting, dialing, disconnected, disconnecting, error, held, holding, incoming, resuming, statechange

    Relacionados con cambios en el DOM

    DOMAttrModified, DOMCharacterDataModified, DOMContentLoaded, DOMElementNameChanged, DOMNodeInserted, DOMNodeInsertedIntoDocument, DOMNodeRemoved, DOMNodeRemovedFromDocument, DOMSubtreeModified

    Relacionados con arrastre de elementos (drag and drop)

    drag, dragend, dragenter, dragleave, dragover, dragstart, drop

    Relacionados con video y audio

    audioprocess, canplay, canplaythrough, durationchange, emptied, ended, ended, loadeddata, loadedmetadata, pause, play, playing, ratechange, seeked, seeking, stalled, suspend, timeupdate, volumechange, waiting, complete

    Relacionados con la conexión a internet

    disabled, enabled, offline, online, statuschange, connectionInfoUpdate

    Otros tipos de eventos

    Hay más tipos de eventos: relacionados con la pulsación sobre pantallas, uso de copy and paste (copiar y pegar), impresión con impresoras, etc.
